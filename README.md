# 📋 タスク整理・思考の整理

週次タスク管理と思考の整理を行うためのリポジトリです。

## 📁 ファイル構成

```
.
├── タスク整理・思考の整理.md    # メインのタスク管理ファイル（週次更新）
├── README.md                    # このファイル
├── .gitignore                   # Git除外設定
├── daily-logs/                  # 日報（デイリーログ）
│   └── 2026-03/                 # 月別フォルダ
│       └── 2026-03-22.md        # 日付ごとの日報ファイル
├── docs/                        # ドキュメント
│   └── git/                     # Git関連ドキュメント
│       ├── Gitセットアップ手順.md
│       ├── Git設定クイックガイド.md
│       ├── GitHub接続手順.md
│       ├── Git_Credential_Manager使い方.md
│       └── プッシュ手順.md
└── templates/                   # テンプレート
    ├── 週次タスク_テンプレート.md
    └── 日報テンプレート.md
```

## 🎯 使い方

### 週の初め
1. 今週の計画セクションに目標とタスクを記入
2. Mermaidガントチャートにタスクを追加
3. 詳細タスク管理テーブルを更新

### 毎日
1. タスクの進捗を更新
2. 新しいタスクを追加
3. 完了したタスクを「完了したタスク」セクションに移動
4. 日報を記入（下記「日報の使い方」参照）

### 週の終わり/翌週初め
1. 週次レビューセクションで振り返り
2. 来週の計画を立てる
3. 必要に応じて新しい週次ファイルを作成

## 📝 日報（デイリーログ）の使い方

`daily-logs/` フォルダに日付ごとのマークダウンファイルを作成して、その日のやったことや気づきを記録します。

### ファイルの作り方

1. `templates/日報テンプレート.md` をコピー
2. `daily-logs/YYYY-MM/YYYY-MM-DD.md` として保存（例: `daily-logs/2026-03/2026-03-22.md`）
3. 日付や曜日を記入して、各セクションを埋める

### 記録する内容

- **やったこと**: その日に完了・着手した作業
- **気づき・学び**: 仕事を通じて気づいたこと
- **困っていること・課題**: 解決が必要な問題
- **明日やること**: 翌日の予定
- **メモ**: その他自由記述

### 運用ルール

- 1日1ファイル（`YYYY-MM-DD.md`形式）
- 月ごとにフォルダを分ける（`daily-logs/YYYY-MM/`）
- 書いたらコミット＆プッシュで履歴を残す

## 📊 機能

- **ガントチャート**: Mermaid形式で週間タスクを視覚化
- **WBS**: 作業分解構造でタスクを階層化
- **優先順位管理**: 緊急度・重要度に基づくタスク分類
- **思考の整理**: 気づき、課題、アイデアを記録
- **週次レビュー**: 振り返りと次週計画
